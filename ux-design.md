# 십삼월 앱 — 항목 선택 부담 해소 UX 설계

---

## 1. 문제 정의

### 현재 상태
| 구분 | 항목 수 | 예시 |
|------|---------|------|
| 비과세 | 3개 | 식대, 자가운전보조금, 출산보육수당 |
| 소득공제 | 4개 | 카드, 주택청약, 전세대출, 주담대 |
| 세액공제 | 6개 | 의료비, 교육비, 기부금, 월세, 연금저축, 보험료 |
| 추가공제 | 4개 | 경로우대, 장애인, 부녀자, 한부모 |
| **합계** | **17개** | |

### 핵심 고통
- 17개 항목을 한 번에 보면 **"이걸 다 해야 해?"** 반응
- 각 항목마다 세부 분류가 또 있음 (교육비 5종, 의료비 4종)
- 자기한테 해당되는 게 뭔지 모름
- 해당 안 되는 항목까지 읽어야 해서 인지 부하 발생
- 결국 "나중에 해야지" → 이탈

### 설계 원칙
1. **한 화면에 하나의 의사결정** — 토스 송금 플로우 참조
2. **안 보여줘도 되는 건 안 보여준다** — 프로필 기반 필터링
3. **돈이 되는 순서대로** — 절세 효과 기준 정렬
4. **언제든 건너뛸 수 있다** — 스킵 부담 제로

---

## 2. 전체 플로우 구조

```
[연봉 입력] → [프로필 4문항] → [맞춤 항목 리스트] → [항목별 입력] → [결과]
```

5단계, 각 단계를 상세히 설계한다.

---

## 3. STEP 1 — 연봉 입력

### 목적
- 소득구간에 따라 공제율, 한도, 절세 효과가 완전히 달라짐
- 이 하나의 숫자로 전체 시뮬레이션의 기반이 됨

### 와이어프레임

```
┌─────────────────────────────────────┐
│                                     │
│  (상단 여백)                         │
│                                     │
│  올해 연봉이 얼마예요?                │
│  세전 총급여 기준이에요               │
│                                     │
│        ┌─────────────────┐          │
│        │   4,800  만원    │          │
│        └─────────────────┘          │
│                                     │
│  급여명세서 '지급합계'를 입력하면      │
│  가장 정확해요                       │
│                                     │
│                                     │
│                                     │
│                                     │
│  ┌─────────────────────────────┐    │
│  │          다음               │    │
│  └─────────────────────────────┘    │
│                                     │
└─────────────────────────────────────┘
```

### UX 디테일
- 숫자 키패드 자동 노출 (만원 단위)
- 입력 즉시 "월 환산 약 400만원" 서브텍스트 표시 — 맞는지 직관적 확인
- 100만원 단위 대략 입력도 허용 — 정확하지 않아도 괜찮다는 톤
- 빈 상태에서 "다음" 비활성

## 4. STEP 2 — 프로필 질문 (4문항)

### 목적
- 17개 항목 중 사용자에게 **해당되는 것만** 걸러내기 위한 최소 질문
- 4개 질문으로 평균 7~9개 항목만 남길 수 있음

### 필터링 로직

| 질문 | 선택지 | 필터링 효과 |
|------|--------|-------------|
| 결혼 여부 | 미혼 / 기혼 | 미혼이면 부녀자공제, 맞벌이 최적화 제거 |
| 자녀 | 없음 / 있음(나이) | 없으면 자녀세액공제, 교육비(자녀) 제거 |
| 주거 형태 | 무주택월세 / 무주택전세 / 1주택자가 / 기타 | 자가면 월세공제 제거, 무주택이면 주담대이자 제거 등 |
| 부양가족 | 없음 / 부모님 / 기타 | 없으면 경로우대, 장애인공제 숨김 |

## 5. STEP 3 — 맞춤 항목 리스트

### 목적
- 프로필 기반으로 필터링된 항목만 표시
- 절세 효과 큰 순서대로 정렬
- 사용자가 "이것만 하면 되는구나" 느끼게

### 필터링 예시

**Case A: 미혼, 자녀 없음, 월세, 부양가족 없음, 연봉 4,800만원**
→ 표시 항목 7개:
1. 신용카드/체크카드 (예상 절세 ~40만원)
2. 월세 세액공제 (예상 절세 ~90만원)
3. 연금저축/IRP (예상 절세 ~99만원)
4. 의료비 (예상 절세 ~조건부)
5. 보험료 (예상 절세 ~18만원)
6. 기부금 (예상 절세 ~조건부)
7. 교육비-본인 (예상 절세 ~조건부)

**Case B: 기혼, 자녀 2명(초등), 자가 1주택, 부모님 부양, 연봉 7,000만원**
→ 표시 항목 12개, 하지만 TOP 5 강조

### 와이어프레임

```

│  ← 뒤로                            │
│                                     │
│  양훈님에게 해당되는                  │
│  절세 항목이에요                     │
│                                     │
│  연봉 4,800만원 기준 · 7개 항목      │
│                                     │
│  ─ 꼭 챙기세요 ─────────────────    │
│                                     │
│  ┌─────────────────────────────┐    │
│  │  연금저축/IRP          99만원 │    │
│  │  아직 안 넣었다면 지금이 기회  │    │
│  └─────────────────────────────┘    │
│                                     │
│  ┌─────────────────────────────┐    │
│  │  월세 세액공제          90만원 │    │
│  │  월세 내고 있으면 거의 확정    │    │
│  └─────────────────────────────┘    │
│                                     │
│  ┌─────────────────────────────┐    │
│  │  신용카드/체크카드      40만원 │    │
│  │  25% 넘겼으면 체크카드로 전환  │    │
│  └─────────────────────────────┘    │
│                                     │
│  ─ 해당되면 챙기세요 ──────────     │
│                                     │
│  ┌─────────────────────────────┐    │
│  │  보험료              최대 18만 │    │
│  └─────────────────────────────┘    │
│  ┌─────────────────────────────┐    │
│  │  의료비              조건부   │    │
│  └─────────────────────────────┘    │
│  ┌─────────────────────────────┐    │
│  │  기부금              조건부   │    │
│  └─────────────────────────────┘    │
│  ┌─────────────────────────────┐    │
│  │  교육비 (본인)        조건부   │    │
│  └─────────────────────────────┘    │
│                                     │
│                                     │
│  ┌─────────────────────────────┐    │
│  │     하나씩 입력하기          │    │
│  └─────────────────────────────┘    │
│                                     │
│  전체 항목 보기 >                    │
│                                     │
└─────────────────────────────────────┘
```

### UX 디테일
- **2단 구분**: "꼭 챙기세요" / "해당되면 챙기세요"
  - "꼭 챙기세요" = 프로필상 거의 확실히 해당 + 절세 효과 큰 항목
  - "해당되면 챙기세요" = 해당 여부 불확실 or 효과 작은 항목
- 각 항목 카드에 **예상 절세 금액** 표시 — 동기 부여의 핵심
- 금액은 연봉 기반 대략 추정 (정확한 건 입력 후 계산)
- "전체 항목 보기" 링크 — 필터링에서 빠진 항목도 확인 가능
- 항목 카드 탭하면 해당 항목 입력 화면으로 진입 가능 (자유 탐색)
- "하나씩 입력하기" 버튼 = 절세 효과 큰 순서대로 순차 가이드 시작

## 10. 인터랙션 원칙 정리

| 원칙 | 적용 |
|------|------|
| 원탭 전환 | 선택지 탭 → 자동 다음 화면 (금액 입력 제외) |
| 제로 스크롤 | 각 화면이 뷰포트 안에 완결 (스크롤 필요 없게) |
| 숫자 우선 | 금액 입력 시 숫자 키패드 자동 노출, 만원 단위 고정 |
| 맥락 제공 | 모든 입력 필드에 한도/공제율/계산 근거 1줄 |
| 안전한 스킵 | "아니요" = 스킵, 언제든 돌아와서 수정 가능 |
| 진행 감각 | 프로그레스 도트 + "n/7" 텍스트 + 입력완료 표시 |
| 결과 선행 | 리스트 화면에서 예상 절세 금액을 미리 보여줌 |

## 11. 항목 수 체감 비교

| | Before | After |
|---|--------|-------|
| 첫 화면에 보이는 항목 | 17개 전부 | 0개 (연봉 입력만) |
| 실제 입력해야 하는 항목 | 17개 | 평균 7~9개 |
| 의사결정 단위 | 항목 + 세부분류 + 금액 동시 | 한 번에 하나 |
| 이탈 시 손실 | 처음부터 다시 | 자동 저장, 이어서 가능 |
| 동기 부여 | 없음 | 예상 절세 금액 실시간 표시 |
