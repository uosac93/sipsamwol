# 십삼월 온보딩 플로우 심층 설계

> 핵심 원칙: "해당 안 되면 바로 스킵" / 최소 터치로 결과 도달

---

## 0. 현황 진단

### 딜레마
| 방향 | 문제 |
|------|------|
| 질문 많이 하면 | 이탈률 급증. 3번째 화면부터 이탈 시작, 5번째에서 50% 이상 이탈 (토스 내부 데이터 기준 유사 플로우) |
| 질문 적게 하면 | 필터링 부족. 해당 안 되는 항목까지 노출되어 "이걸 다 해야 해?" 반응 |
| 한 화면에 많이 넣으면 | 인지 부하. 읽기 전에 스크롤부터 하면 이탈 |
| 한 화면에 하나만 넣으면 | 화면 수 폭발. 40개 항목 = 40개 화면 |

### 핵심 인사이트
질문의 목적은 "정보 수집"이 아니라 **"항목 제거"**다. 한 질문이 많은 항목을 제거할수록 효율적이다.

---

## 1. 질문별 필터링 파워 분석

각 질문이 제거할 수 있는 공제 항목 수를 정량화했다.

### Tier 1: 메가 필터 (1개 질문으로 5개 이상 제거)

| 질문 | 특정 답변 | 제거되는 항목 | 제거 수 |
|------|----------|-------------|---------|
| 가족 상황 (결혼+자녀 합산) | 미혼+자녀없음 | 배우자공제, 부녀자공제, 자녀세액공제, 교육비(자녀), 보육수당, 산후조리원(의료비), 출산/입양세액공제, 한부모공제 | **8개** |
| 주거 상황 (주택수+거주형태 합산) | 2주택 이상 | 주택청약, 주담대이자, 전세대출, 월세세액공제, 장기주택저당차입금 | **5개** |

### Tier 2: 중간 필터 (1개 질문으로 2~4개 제거)

| 질문 | 특정 답변 | 제거되는 항목 | 제거 수 |
|------|----------|-------------|---------|
| 부양가족 | 없음 | 인적공제(추가), 경로우대, 장애인공제 | **3개** |
| 자녀 나이 (자녀 있을 때만) | 대학생 이상 | 보육수당, 어린이집교육비, 자녀세액공제(일부) | **3개** |

### Tier 3: 마이너 필터 (1개 질문으로 1개 제거)

| 질문 | 특정 답변 | 제거되는 항목 | 제거 수 |
|------|----------|-------------|---------|
| 차량 소유 | 없음 | 자가운전보조금 | **1개** |
| 기부 경험 | 없음 | 기부금 세액공제 | **1개** |

### Tier 0: 질문 불필요 (대부분 해당)

다음 항목은 **질문 없이 기본 노출**해야 한다. 직장인 80% 이상이 해당하기 때문.

- 신용카드/체크카드 소득공제
- 의료비 세액공제
- 보험료 세액공제
- 연금저축/IRP 세액공제
- 교육비(본인) 세액공제

이 항목들은 "해당 여부"를 묻지 않고, **결과 화면에서 금액만 입력**하게 한다.

## 2. 질문 합치기 설계

### 합치기 원칙
- 논리적으로 연결된 질문은 하나의 "생활 상황"으로 합친다
- 선택지는 **최대 4개** (토스 기준: 한 화면에 선택지 4개까지는 스캔 가능)
- 선택지 문구는 "상태"가 아니라 "나의 상황"으로 쓴다

### 합성 질문 1: 가족 상황

**"가족 상황이 어떻게 되세요?"**

| 선택지 | 내부 처리 |
|--------|----------|
| 혼자 살아요 | 미혼 + 자녀없음. 배우자/자녀/부녀자/한부모 관련 8개 항목 전부 OFF |
| 배우자가 있어요 | 기혼 + 자녀없음. 부녀자공제 ON, 맞벌이 최적화 ON. 자녀 관련 OFF |
| 배우자 + 자녀가 있어요 | 기혼 + 자녀있음. 후속 질문으로 자녀 수/나이 1개 추가 |
| 한부모예요 | 한부모공제 자동 ON, 자녀 관련 ON. 후속 질문으로 자녀 수/나이 |

**필터링 효과**: 미혼 선택 시 8개 항목 즉시 제거. 전체 사용자의 약 45%가 미혼이므로, 거의 절반의 사용자가 첫 질문에서 8개를 날릴 수 있다.

**후속 질문 (자녀 있을 때만 노출)**:

"자녀가 몇 명이고, 막내가 몇 살이에요?"
- 숫자 입력 2개: [자녀 수] 명, 막내 [  ] 세
- 이 정보로 보육수당, 어린이집 교육비, 자녀세액공제 금액을 자동 계산

### 합성 질문 2: 주거 상황

**"지금 어떻게 살고 계세요?"**

| 선택지 | 내부 처리 |
|--------|----------|
| 월세 살아요 | 무주택 + 월세. 월세공제 ON, 전세대출 OFF, 주담대 OFF |
| 전세예요 | 무주택 + 전세. 전세대출 ON, 월세공제 OFF, 주담대 OFF |
| 내 집이 있어요 | 1주택. 주담대 ON, 주택청약 조건부, 월세/전세 OFF |
| 집이 2채 이상이에요 | 다주택. 주택 관련 5개 전부 OFF |

**필터링 효과**: 다주택 선택 시 5개 즉시 제거. 1주택이면 월세+전세 2개 제거. 월세면 전세+주담대 2개 제거.

### 합성 질문 3: 부양가족

**"부양하는 가족이 있으세요?"**

| 선택지 | 내부 처리 |
|--------|----------|
| 없어요 | 인적공제 추가분, 경로우대, 장애인공제 OFF |
| 부모님 (만 60세 이상) | 경로우대 ON, 인적공제 ON. 의료비 몰아주기 안내 활성 |
| 기타 (조부모, 형제자매 등) | 인적공제 ON, 세부 확인은 결과 화면에서 |

## 3. "나머지" 항목 처리 전략

### 질문으로 필터링이 불가능한 항목들

| 항목 | 이유 | 처리 방식 |
|------|------|----------|
| 카드 사용액 | 직장인 99% 해당 | **무조건 표시**, 금액만 입력 |
| 의료비 | 직장인 80% 이상 해당 | **무조건 표시**, 금액만 입력 |
| 보험료 | 직장인 70% 이상 해당 | **무조건 표시**, 금액만 입력 |
| 연금저축/IRP | 가입 여부 불확실하지만 효과 큼 | **무조건 표시**, 미가입자에게는 "가입하면 이만큼 절세" 안내 |
| 교육비(본인) | 대학원/학점은행 등 해당자 적음 | 결과 화면에서 "혹시 이것도?" 유도 |
| 기부금 | 해당자 적지만 고향사랑기부제는 전원 가능 | 결과 화면에서 "10만원 기부하면 전액 환급" 유도 |

### 핵심 원칙
> 직장인 과반수가 해당하는 항목은 **질문하지 않는다**. 기본 노출하고 금액만 받는다.
> 소수만 해당하는 항목은 프로필 질문으로 **걸러내거나**, 결과 화면에서 **추천**한다.

## 4. 궁극의 심플: 질문 제로 루트

### 간편인증 연동 시나리오

```
화면1: 연봉 입력 (또는 간편인증으로 자동)
화면2: 간편인증 동의 (카카오/PASS/네이버)
화면3: 결과 (홈택스 데이터 기반 자동 계산)
```

**장점**: 2~3단계로 끝. 질문 0개. 입력 0개. 이탈률 최저.

**단점**:
- 간편인증 연동 개발 비용 높음 (홈택스 스크래핑 or 마이데이터)
- 1~9월에는 홈택스에 올해 데이터가 없음 (10월 미리보기 이후에만 유효)
- 초기 MVP에서는 구현 불가능할 수 있음

**결론**: v2 이후 킬러 기능으로 두되, v1에서는 수동 입력 기반으로 설계.

### 하이브리드 시나리오 (v1.5)

```
화면1: 연봉 입력
화면2: "간편인증으로 바로 볼래요" / "직접 입력할게요"
  - 간편인증 → 화면3: 결과
  - 직접 입력 → 프로필 질문 → 결과
```

간편인증이 되면 프로필 질문을 **전부 스킵**한다. 홈택스 데이터에 가족관계, 주택 정보, 공제 내역이 다 있기 때문.

## 5. 체크리스트 vs 개별 질문 vs 하이브리드

### Option A: 순수 개별 질문 (현재 ux-design.md 안)

```
화면1: 연봉 → 화면2: 가족 → 화면3: 주거 → 화면4: 부양가족 → 화면5: 결과
```

| 항목 | 평가 |
|------|------|
| 총 화면 수 | 5개 (자녀 있으면 6개) |
| 터치 수 | 5~6회 |
| 토스 적합도 | 높음 (한 화면 한 질문) |
| 필터링 정확도 | 높음 |
| 체감 속도 | 보통 (화면 전환 4~5회) |

### Option B: 순수 체크리스트

```
화면1: 연봉
화면2: 해당하는 것에 체크하세요
  [ ] 결혼했어요
  [ ] 자녀가 있어요
  [ ] 부모님을 부양해요
  [ ] 집이 있어요
  [ ] 월세 살아요
  [ ] 전세예요
화면3: 결과
```

| 항목 | 평가 |
|------|------|
| 총 화면 수 | 3개 |
| 터치 수 | 최소 1회 ~ 최대 6회 |
| 토스 적합도 | **낮음** (토스는 체크리스트를 거의 안 씀) |
| 필터링 정확도 | 낮음 (상호배타 옵션을 처리 못함 - 월세/전세/자가는 동시선택 불가인데 체크리스트로는 이걸 표현 못함) |
| 체감 속도 | 빠름 (화면 전환 2회) |

**문제점**:
- 월세/전세/자가가 동시에 체크 가능해져서 데이터 오류 발생
- "집이 있어요"와 "월세 살아요"를 동시에 체크하면 어떻게 처리할 것인가
- 체크리스트는 "내가 뭘 체크해야 하지?"라는 인지 부하를 유발
- 하나도 체크 안 한 사람은 "다 해당 안 되나?" 불안감

### Option C: 하이브리드

```
화면1: 연봉
화면2: 가족 상황 (4지선다, 한 화면)
화면3: 주거 상황 (4지선다, 한 화면)
화면4: 빠른 체크 — 나머지
  [부양가족 있어요] [차량 있어요]
화면5: 결과
```

| 항목 | 평가 |
|------|------|
| 총 화면 수 | 5개 (자녀 있으면 6개) |
| 터치 수 | 4~5회 |
| 토스 적합도 | 중간 (핵심은 개별 질문, 마이너는 체크) |
| 필터링 정확도 | 높음 (핵심 질문은 상호배타 선택지) |
| 체감 속도 | 보통 |

**평가**: Option A와 화면 수가 같아서 하이브리드의 이점이 약함. 차량/부양가족을 체크리스트로 합치면 1화면 줄지만, UX 일관성이 깨짐.

## 6. 이탈률 시뮬레이션

토스/핀테크 온보딩 퍼널 데이터 기반 추정 (화면당 이탈률 5~12%):

### 시나리오별 예상 완주율

| 시나리오 | 화면 수 | 화면당 이탈률 | 최종 완주율 |
|----------|---------|-------------|-----------|
| A. 개별 질문 5화면 | 5 | 7% | **69%** |
| A+. 개별 질문 6화면 (자녀 후속) | 6 | 7% | **64%** |
| B. 체크리스트 3화면 | 3 | 8%* | **78%** |
| C. 하이브리드 5화면 | 5 | 7% | **69%** |
| D. 간편인증 2화면 | 2 | 5% | **90%** |
| E. 스마트 2+1화면 (아래 추천안) | 3 | 6% | **83%** |

*체크리스트는 화면 수는 적지만, 체크리스트 자체의 인지 부하로 화면당 이탈률이 약간 높음

## 7. 최종 추천안 3가지

---

### 추천안 1: "스마트 3화면" (강력 추천)

**컨셉**: 메가 필터 2개만 화면으로, 나머지는 자동 처리

```
화면1: 연봉 입력
화면2: 가족 상황 (4지선다)
  → 미혼 선택 시 8개 항목 즉시 제거
  → 자녀 있음 선택 시 하단에 자녀수/나이 인라인 확장 (화면 전환 없음)
화면3: 주거 상황 (4지선다)
  → 다주택 선택 시 5개 항목 즉시 제거
화면4: 맞춤 결과 리스트 + 금액 입력 시작
```

**부양가족은 어떻게?**
- 화면2에서 "부모님 부양" 선택지를 가족 상황에 통합하지 않는다 (선택지가 너무 많아짐)
- 대신 결과 리스트에서 "혹시 부양하는 가족이 있으세요?" 배너를 띄운다
- 탭하면 인적공제/경로우대 항목이 추가됨

**차량은?**
- 자가운전보조금은 해당자가 적고 금액도 작아서 묻지 않는다
- 결과 화면 하단 "더 챙길 수 있는 항목"에서 노출


#### 평가

| 항목 | 내용 |
|------|------|
| 화면 수 | 2~3개 (연봉 + 가족 + 결과) |
| 터치 수 | 2~4회 |
| 예상 완주율 | **87%** |
| 토스 적합도 | **중간** -- 결과 화면에 칩 필터가 들어가면서 토스 원칙(한 화면 한 의사결정)에서 살짝 벗어남 |
| 필터링 정확도 | **중간** -- 가족 상황만 사전 필터링. 주거/부양은 결과에서 사후 조정 |
| 장점 | 온보딩 최단 경로. 완주율 최고. "빨리 결과 보여주고, 결과에서 조정" 철학 |
| 단점 | 결과 화면이 복잡해짐. 칩 필터 + 배너 + 항목 리스트가 한 화면에 있으면 초기 인지 부하 발생. 디폴트 상태에서 주택 관련 항목 4~5개가 불필요하게 노출될 수 있음 |
| 적합 상황 | "일단 빨리 결과 보여주는 게 최우선"일 때. 사용자가 결과를 보면서 자기 상황을 맞추는 게 더 자연스러울 때 |

## 9. 최종 권고

### v1 MVP: 추천안 1 "스마트 3화면"

이유:
1. **완주율과 필터링의 최적 밸런스** -- 83% 완주율이면 충분히 높고, 메가 필터 2개로 핵심 필터링 커버
2. **토스 원칙에 충실** -- 한 화면 한 질문, 선택 즉시 다음으로
3. **구현 복잡도 적절** -- 칩 필터 같은 복잡한 UI 없이, 단순 선택지 + 인라인 확장으로 해결
4. **부양가족은 결과 배너로 충분** -- 부양가족 해당자(약 30%)만 배너 탭하면 됨. 70%는 이 질문을 안 봐도 됨

### v1.5: 간편인증 루트 추가

- 화면2에 "간편인증으로 바로 보기" 옵션 추가
- 인증하면 프로필 질문 전부 스킵 → 바로 결과
- 10월 이후(홈택스 미리보기 오픈)에 특히 효과적

### v2: 추천안 2로 전환 검토

- 사용자 데이터 쌓이면 부양가족 배너 클릭률 분석
- 클릭률 낮으면(= 해당자가 놓치고 있으면) → 별도 화면으로 분리하는 추천안 2 적용

## 10. 전체 사용자 여정 (추천안 1 기준)

```
[앱 진입]
    │
    ▼
[화면1] 연봉 입력
    │   숫자 키패드, "월 환산 약 400만원" 확인

    ▼
[화면2] 가족 상황
    │   "혼자 살아요" / "배우자 있어요" / "배우자+자녀" / "한부모"
    │   터치 1회 (선택 즉시 다음)
    │   ※ 자녀 있으면 인라인으로 수/나이 추가 (터치 2~3회 더)
    ▼
[화면3] 주거 상황
    │   "월세" / "전세" / "자가" / "2채+"
    │   터치 1회 (선택 즉시 다음)
    ▼
[화면4] 맞춤 결과 리스트
    │   프로필 기반 필터링된 항목만 표시
    │   "꼭 챙기세요" / "해당되면 챙기세요" 2단 구분
    │   상단에 "부양가족 있으면 탭하세요" 배너
    │   "하나씩 입력하기" CTA
    ▼
[항목별 입력] 순차 가이드
    │   절세 효과 큰 순서대로
    │   해당 안 되면 "아니요" 한 번에 스킵
    ▼
[결과] 예상 환급액
    │   금액 + 계산 근거 + "더 받는 방법"
    ▼
[저장] → 연중 트래킹으로 연결
```

**미혼 + 무주택 월세 사용자 최단 경로**:
- 연봉 입력 → "혼자 살아요" → "월세 살아요" → 결과 (7개 항목만 표시)
- **터치 4회, 화면 4개, 소요 시간 약 15초**

**기혼 + 자녀2 + 자가 + 부모 부양 사용자 최장 경로**:
- 연봉 입력 → "배우자+자녀" → 자녀 2명, 막내 5세 → "자가" → 결과 → 부양가족 배너 탭
- **터치 7~8회, 화면 4개 + 인라인 1개, 소요 시간 약 30초**

## 11. 필터링 로직 상세 (구현용)

### 입력 변수

```
salary: number          // 총급여 (만원)
familyStatus: enum      // SINGLE | MARRIED | MARRIED_WITH_CHILD | SINGLE_PARENT
childCount: number      // 자녀 수 (0~)
youngestChildAge: number // 막내 자녀 나이
housingStatus: enum     // RENT | JEONSE | OWN_1 | OWN_2_PLUS
hasDependents: boolean  // 부양가족 여부 (결과 화면에서 설정)
dependentType: enum     // NONE | PARENT | OTHER
```

### 필터링 룰

```
// 가족 상황 필터링
if (familyStatus === SINGLE) {
  HIDE: 배우자공제, 부녀자공제, 자녀세액공제, 교육비(자녀),
        보육수당, 산후조리원, 출산입양세액공제, 한부모공제,
        맞벌이최적화
}

if (familyStatus === MARRIED) {
  HIDE: 자녀세액공제, 교육비(자녀), 보육수당, 한부모공제
  SHOW: 부녀자공제(조건부), 맞벌이최적화
}

if (familyStatus === MARRIED_WITH_CHILD) {
  SHOW: 자녀세액공제, 교육비(자녀), 부녀자공제, 맞벌이최적화
  if (youngestChildAge <= 6) SHOW: 보육수당
  if (youngestChildAge <= 5) SHOW: 어린이집교육비
}

if (familyStatus === SINGLE_PARENT) {
  SHOW: 한부모공제, 자녀세액공제, 교육비(자녀)
  HIDE: 배우자공제, 부녀자공제
}

// 주거 상황 필터링
if (housingStatus === RENT) {
  SHOW: 월세세액공제, 주택청약
  HIDE: 주담대이자, 전세대출
}

if (housingStatus === JEONSE) {
  SHOW: 전세대출, 주택청약
  HIDE: 주담대이자, 월세세액공제
}

if (housingStatus === OWN_1) {
  SHOW: 주담대이자
  HIDE: 월세세액공제, 전세대출
  CONDITIONAL: 주택청약 (무주택 세대원이면 가능)
}

if (housingStatus === OWN_2_PLUS) {
  HIDE: 주담대이자, 월세세액공제, 전세대출, 주택청약, 장기주택저당
}

// 부양가족 필터링 (결과 화면에서)
if (!hasDependents) {
  HIDE: 인적공제(추가), 경로우대, 장애인공제
}

if (dependentType === PARENT) {
  SHOW: 경로우대(만70세이상), 인적공제, 의료비(부양가족분)
}

// 연봉 기반 조건부
if (salary > 7000) {
  HIDE: 월세세액공제 (총급여 7천만원 초과 시 불가)
}

if (salary > 5000) {
  부녀자공제 불가 (종합소득 3천만원 = 총급여 약 4,147만원이지만 안전마진)
}
```

### 기본 노출 항목 (질문 없이 항상 표시)

```
ALWAYS_SHOW: [
  신용카드/체크카드,
  의료비(본인),
  보험료,
  연금저축/IRP
]
```

## 12. 마이크로카피 가이드

온보딩 전체에서 사용하는 톤앤매너:

| 상황 | 나쁜 예 | 좋은 예 |
|------|---------|---------|
| 질문 | "결혼 여부를 선택하세요" | "가족 상황이 어떻게 되세요?" |
| 부연 | "맞춤형 공제 항목을 추천합니다" | "맞춤 절세 항목을 골라드릴게요" |
| 선택지 | "미혼" | "혼자 살아요" |
| 선택지 | "기혼, 자녀 있음" | "배우자 + 자녀가 있어요" |
| 스킵 유도 | "건너뛰기" (별도 버튼) | "아직이요" / "없어요" (선택지 자체가 스킵) |
| 안심 | "정확한 금액을 입력하세요" | "대략 넣어도 괜찮아요" |
| 결과 연결 | "온보딩이 완료되었습니다" | (아무 말 없이 결과 화면으로 슬라이드) |

### 톤 원칙
1. **존댓말이되 딱딱하지 않게** -- "~하세요"보다 "~이에요?"
2. **상태가 아니라 상황으로** -- "미혼"이 아니라 "혼자 살아요"
3. **스킵에 죄책감 없게** -- "건너뛰기"가 아니라 "없어요"가 곧 스킵
4. **전문용어 제거** -- "소득공제"가 아니라 "절세", "세액공제"가 아니라 "환급"
