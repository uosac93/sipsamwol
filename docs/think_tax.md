## 1. 각 질문의 필터링 파워

### 정의
- **필터링 파워** = 특정 답변으로 제거(비활성화)할 수 있는 항목 수
- 높을수록 "가성비 좋은 질문"

### 질문별 분석

| 순위 | 질문 | 핵심 답변 | 제거 항목 | 제거 수 | 비고 |
|:---:|------|----------|----------|:---:|------|
| **1** | 가족 상황 (결혼+자녀 합산) | "미혼, 자녀 없음" | #3 배우자공제, #5 자녀공제, #9 부녀자, #10 한부모, #22 자녀세액, #23 출생/입양, #28 산후조리원, #31 교육비(취학전), #32 교육비(초중고), #33 교육비(대학), #38 보육수당, #40 출산지원금 | **12개** | 전체의 30% |
| **2** | 주거 상황 (주택수+거주형태) | "2주택 이상" | #14 전세대출, #15 주담대, #16 주택청약, #35 월세세액공제 | **4개** | 주거 관련 전멸 |
| | | "1주택 자가" | #14 전세대출, #35 월세세액공제, #16 주택청약(조건부) | **3개** | |
| | | "월세" | #14 전세대출, #15 주담대 | **2개** | |

### 핵심 발견

> **가족 상황 1개 질문이 최대 12개 항목을 제거한다.** 이것이 압도적 1순위 질문이다.
> 주거 상황은 2~4개 제거로 2순위.
> 부양가족은 4개 제거로 3순위.
> 나머지는 질문 대비 효과가 미미하다.

## 2. 질문 간 상관관계와 연쇄 스킵

### 상관관계 매트릭스

```
결혼 여부 ──→ 자녀 유무 (미혼이면 자녀 질문 스킵 가능성 높음*)
         └──→ 부녀자공제 (미혼 남성이면 자동 OFF)
         └──→ 결혼세액공제 (미혼이면 OFF)

자녀 유무 ──→ 자녀 나이 (자녀 없으면 스킵)
         └──→ 보육수당 (자녀 없으면 OFF)
         └──→ 출산지원금 (자녀 없으면 OFF)
         └──→ 교육비 자녀분 전부 (자녀 없으면 OFF)

주택 2채+ ──→ 주택청약 (OFF)
          └──→ 전세대출 (OFF)
          └──→ 주담대 (OFF)
          └──→ 월세세액공제 (OFF)

부양가족 없음 ──→ 경로우대 (OFF)
             └──→ 직계존속/형제자매 공제 (OFF)
             └──→ 부양가족 의료비 (OFF)
```

*미혼이어도 자녀가 있을 수 있다 (한부모). 따라서 "혼자 살아요"와 "한부모예요"를 별도 선택지로 둬야 한다.

### 연쇄 스킵 시나리오

| 사용자 유형 | 질문 흐름 | 연쇄 제거 항목 수 |
|------------|----------|:---:|
| 미혼 + 다주택 | 가족→미혼 (12개 OFF) → 주거→다주택 (4개 OFF) | **16개** |
| 미혼 + 월세 | 가족→미혼 (12개 OFF) → 주거→월세 (2개 OFF) | **14개** |
| 기혼+자녀 + 1주택 | 가족→자녀있음 (0개 OFF) → 자녀나이 입력 → 주거→자가 (3개 OFF) | **3개** |
| 미혼 + 전세 + 부양가족 없음 | 가족→미혼 (12개) → 주거→전세 (2개) → 부양가족→없음 (4개) | **18개** |

> **최악의 시나리오**: 기혼+자녀+부양가족+월세 → 거의 모든 항목이 살아남아 제거가 적음 (3개 이하)
> **최선의 시나리오**: 미혼+다주택+부양가족없음 → 20개 항목 제거 (전체의 50%)

### 유형별 최적 플로우

**1위 유형 (미혼+무주택+월세, ~25%)**
- 가족 질문 → "혼자 살아요" → 12개 제거
- 주거 질문 → "월세 살아요" → 2개 추가 제거
- **2개 질문으로 14개 제거** (전체의 35%)
- 남는 핵심 항목: 카드, 의료비, 보험료, 연금저축, 월세공제, 주택청약
- 매우 깔끔한 결과 화면 제공 가능

**3위 유형 (기혼+유자녀+1주택, ~18%)**
- 가족 질문 → "배우자+자녀 있어요" → 제거 없음
- 자녀 나이 입력 → 자동 분기 (보육수당/자녀세액공제/교육비)
- 주거 질문 → "내 집 있어요" → 월세/전세 2개 제거
- **3개 질문으로 2~3개 제거** (제거 적지만, 자녀 나이로 세부 분기가 중요)
- 이 유형은 "항목 제거"보다 "항목 내 금액 최적화"가 핵심

| 자동 제거 항목 | 12개 (연봉 기반 1~3개 추가) |
| 사용자에게 노출 항목 | 25~28개 |
| 필터링률 | **30~35%** |
| 문제 | 해당 안 되는 항목이 너무 많이 노출됨. "이걸 다 해야 해?" 반응 |

#### 질문 1개: 가족 상황

| 지표 | 값 |
|------|-----|
| 추가 제거 (미혼 시) | +12개 |
| 추가 제거 (기혼무자녀 시) | +5개 |
| 추가 제거 (기혼유자녀 시) | +0개 (자녀 나이로 세부 분기) |
| **가중평균 추가 제거** | **+6.4개** |
| 누적 필터링률 | **46%** (미혼이면 60%) |

#### 질문 2개: 가족 상황 + 주거 상황

| 지표 | 값 |
|------|-----|
| 추가 제거 (다주택 시) | +4개 |
| 추가 제거 (1주택 시) | +3개 |
| 추가 제거 (월세 시) | +2개 |
| 추가 제거 (전세 시) | +2개 |
| **가중평균 추가 제거** | **+2.5개** |
| **누적 필터링률** | **52~55%** |
| 미혼+월세 케이스 | **65%** (26개 중 17개 제거) |

#### 질문 3개: 가족 + 주거 + 부양가족

| 지표 | 값 |
|------|-----|
| 추가 제거 (부양가족 없음 시) | +4개 |
| 추가 제거 (부양가족 있음 시) | +0개 |
| **가중평균 추가 제거** | **+3.2개** (80%가 부양가족 없음) |
| **누적 필터링률** | **60~63%** |
| 미혼+월세+부양가족없음 | **75%** (30개 제거) |

#### 질문 4개: 가족 + 주거 + 부양가족 + 차량

| 지표 | 값 |
|------|-----|
| 추가 제거 | +1개 (차량 없을 때) |
| **누적 필터링률** | **62~65%** |
| **한계효용** | **급격히 감소** — 1개 질문으로 겨우 1개 항목 제거 |

### 한계효용 곡선

```
필터링률
  75% ┤                          ◆ (미혼+월세+부양X)
      │
  65% ┤                    ●───────● (4개 질문)
      │                  /
  60% ┤              ●──           (3개 질문)
      │            /
  55% ┤        ●──                 (2개 질문)
      │      /
  46% ┤  ●──                       (1개 질문)
      │  /
  33% ┤●                           (0개 질문, 연봉만)
      │
      └───┬───┬───┬───┬───┬───
          0   1   2   3   4   5  질문 수
```

### 최적점 분석

| 질문 수 | 필터링률 | 추가 질문 1개당 필터링 증가분 | 이탈률 영향 | 판정 |
|:---:|:---:|:---:|:---:|:---:|
| 0개 | 33% | - | 없음 | 필터링 부족 |
| 1개 | 46% | +13%p | 낮음 | 부족 |
| **2개** | **55%** | **+9%p** | **낮음** | **최소 권장** |
| **3개** | **62%** | **+7%p** | **보통** | **최적** |
| 4개 | 65% | +3%p | 높아짐 | 과잉 |
| 5개+ | 67% | +2%p | 매우 높음 | 비효율 |

## 6. 최종 추천

### 추천안: 질문 3개 (연봉 포함 시 4화면)

```
화면 1: 연봉 입력              → 연봉 기반 자동 분기 3개 항목
화면 2: 가족 상황 (4지선다)     → 최대 12개 제거
  └─ 자녀 있음 시: 인라인으로 자녀수/막내나이 → 자녀 세부 자동 분기
화면 3: 주거 상황 (4지선다)     → 최대 4개 제거
화면 4: 부양가족 유무 (2지선다)  → 최대 4개 제거
화면 5: 결과
```

### 이 구조의 성과

| 지표 | 수치 |
|------|------|
| 총 질문 수 | 3개 (연봉 제외) |
| 총 화면 수 | 5개 (결과 포함) |
| 터치 수 | 4~6회 |
| 평균 필터링률 | **62%** |
| 미혼 사용자 필터링률 (45%) | **70~75%** |
| 예상 완주율 | **78~83%** |

### 결과 화면에서 잡는 나머지

질문으로 커버하지 않는 항목은 결과 화면에서 처리한다:

| 항목 | 결과 화면 처리 방식 |
|------|-------------------|
| #8 장애인공제 | "혹시 이것도?" 섹션에서 노출 |
| #18 개인연금저축 (2000년 이전) | 해당자 극소수, 노출 안 해도 무방 |
| #19 우리사주 | "혹시 이것도?" 섹션 |
| #20 장기집합투자증권 | 2015년 이전 가입자만, 노출 안 해도 무방 |
| #29 난임시술비 | "혹시 이것도?" 섹션 |
| #34 기부금 | "고향사랑기부제 10만원 기부 → 전액 환급" 배너 |
| #36 결혼세액공제 | 기혼 선택 시 자동 안내 |
| #37 중소기업 취업자 감면 | "혹시 이것도?" 섹션 (향후 회사 정보 연동 시 자동) |
| #39 자가운전보조금 | "혹시 이것도?" 하단 |

### 왜 질문 2개가 아니라 3개인가

- 질문 2개(가족+주거)만으로도 미혼 사용자에게는 충분 (65% 필터링)
- 그러나 **기혼 사용자의 경우 가족 질문에서 제거되는 항목이 0개**이므로, 주거만으로는 55%에 그침
- 부양가족 질문 1개를 추가하면 기혼 사용자의 필터링률이 +7%p 상승
- 부양가족 질문은 2지선다(있다/없다)로 **1초 만에 답변 가능** → 이탈 위험 극히 낮음

### 왜 질문 4개가 아니라 3개인가

- 4번째 질문(차량)은 1개 항목만 제거 → ROI 최저
- 화면 수가 6개가 되면 이탈률이 급증하는 구간에 진입
- 차량 정보는 결과 화면에서 간단히 처리 가능

## 부록: 질문 설계 시 세무적 주의사항

### 가족 상황 질문에서의 함정

1. **"혼자 살아요"의 정의를 명확히 해야 함**
   - 세법상 "1인 세대"와 "미혼"은 다름
   - 부모님과 동거하는 미혼자는 "혼자 살아요"를 선택하면 부양가족 공제가 누락될 수 있음
   - 해결: 부양가족 질문을 별도로 두되, "혼자 살아요" 선택자에게도 물어야 함

2. **한부모 판정**
   - 세법상 한부모 = 배우자 없이 기본공제 대상 직계비속이 있는 경우
   - 이혼 후 자녀 양육권이 없으면 한부모공제 불가
   - 사실혼 관계는 배우자 없음으로 판단

3. **결혼세액공제는 "혼인신고 연도"가 기준**
   - 2024~2026년에 혼인신고한 경우만 해당
   - "기혼"이라고 답했다고 자동 적용하면 안 됨 → 결과 화면에서 "올해 혼인신고 하셨나요?" 확인

### 주거 상황 질문에서의 함정

1. **세대주 vs 세대원**
   - 전세대출 공제, 주택청약 공제, 월세세액공제 모두 세대주 요건
   - 세대주가 주택 관련 공제를 받지 않으면 세대원도 가능
   - 온보딩에서 이 복잡성을 물으면 이탈 → 결과 화면에서 세대주 여부 확인

2. **"내 집이 있어요"의 기준**
   - 세법상 1주택은 "세대 기준"이지 개인 기준이 아님
   - 배우자 명의 주택도 세대 주택 수에 포함
   - 온보딩에서는 "우리 집(세대) 기준으로 집이 몇 채인가요?"로 물어야 함

### 연봉 자동 판단의 한계

1. **총급여 vs 연봉**
   - 사용자가 입력하는 "연봉"과 세법상 "총급여"는 다름
   - 총급여 = 연봉 - 비과세소득 (식대, 자가운전보조금 등)
   - 비과세를 감안하면 연봉 7,500만 이하가 대략 총급여 7,000만 이하
   - 경계 구간 사용자에게는 정확한 총급여 확인이 필요
