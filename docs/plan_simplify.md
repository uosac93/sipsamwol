## 1. 현황 진단

### 현재 구조의 문제점

| 문제 | 구체적 상황 | 사용자 영향 |
|------|-----------|-----------|
| 항목 과다 | ALL_DEDUCTION_ITEMS 배열에 35+개 항목 (비과세 12종, 소득공제 12종, 세액공제 11종) | 스크롤 지옥, 이탈 |
| 아코디언 중첩 | 5개 카테고리(인적공제/특별소득/카드등소득/기타소득/세액공제) 각각 아코디언으로 펼침 | 어디에 뭐가 있는지 모름 |
| 세부 입력 과다 | 카드 7종, 의료비 6종, 교육비 7종, 기부금 3+2종 | 하나만 입력하려고 왔는데 압도당함 |
| 전문용어 남발 | "특별소득공제", "세액공제", "비과세", "근로제공기간 월할" | 뭔 말인지 모름 |
| 네비게이션 불일관 | 온보딩은 back-btn, 탭은 tab-bar, 시트는 overlay, 설정은 save-btn | 뒤로가기 혼란 |
| 고급 기능 동급 노출 | 중도입사/퇴사, 복수 근무처, 외국인 단일세율, 전 직장 원천징수영수증이 설정 탭에 바로 보임 | 대부분 해당 안 되는데 복잡해 보임 |

## 2. 핵심 원칙 (토스식)

1. **한 화면에 하나의 의사결정** - 한 번에 하나만 묻고, 하나만 입력하게
2. **복잡함은 숨기고, 필요할 때만 꺼내기** - 기본은 미니멀, 확장은 선택적
3. **기본값으로 80% 해결** - 대부분 직장인은 5~6개 항목만 해당
4. **전문용어 제거** - 세법 용어 대신 일상 언어
5. **뒤로가기 항상 가능** - 어디서든 이전 상태로 복귀

## 3. 3단계 레이어 구조

### 레이어1: 기본 (대부분의 직장인)

프로필 기반으로 **최대 6개만** 기본 표시.

| 순서 | 항목 | 현재 id | 표시 조건 |
|------|------|---------|----------|
| 1 | 카드 사용액 | `card` | 항상 (총액 하나만) |
| 2 | 의료비 | `medical` | 항상 (총액 하나만) |
| 3 | 연금저축/IRP | `pension` | 항상 |
| 4 | 보장성보험료 | `insurance` | 항상 (총액 하나만) |
| 5 | 월세 | `rent` | 무주택+월세 거주 시 |
| 6 | 부양가족 | `dependents` | 부양가족 있을 때 |

**디자인**: 카드형 리스트. 각 항목은 한 줄 입력 + 토글. 아코디언 아님.

### 레이어2: 확장 ("더 많은 공제 보기")

레이어1 하단에 **"더 많은 공제 보기"** 버튼. 누르면 나머지 항목이 카테고리 없이 flat list로 표시.

포함 항목:
- 교육비, 기부금(법정/지정), 정치자금 기부금, 고향사랑 기부금
- 주택청약, 주택담보대출, 전세자금대출
- 결혼세액공제, 자녀세액공제
- 중소기업 소득세 감면
- 한부모/부녀자 추가공제, 경로우대, 장애인 추가공제

**디자인**: 검색 가능한 리스트. 각 항목 옆에 "해당 없음" / "입력하기" 2택.

### 레이어3: 상세 (세부 분류)

레이어1~2의 특정 항목을 탭하면 세부 분류 화면으로 진입.

| 항목 | 기본 입력 | 상세 입력 (탭 시) |
|------|----------|-----------------|
| 카드 | "총 카드 사용액" 1개 | 신용/체크/현금/전통시장/대중교통/문화/영화 7종 |
| 의료비 | "총 의료비" 1개 | 일반/본인65세장애인/난임/미숙아/안경/산후조리 6종 |
| 교육비 | "총 교육비" 1개 | 본인/취학전/초중고/대학/장애인특수/교복/학자금상환 7종 |
| 기부금 | "총 기부금" 1개 | 법정/지정(비종교)/지정(종교) 3종 |
| 보험료 | "총 보험료" 1개 | 일반보장성/장애인전용 2종 |

**진입 방식**: 기본 입력 필드 하단에 "상세 입력 >" 텍스트 버튼. 탭하면 바텀시트 or 새 화면.

## 4. 입력 심플화 전략

### 4-1. 총액 우선 입력

**현재**: 카드 공제 열면 바로 7개 input이 보임
**변경**:

```
[카드 사용]
  총 사용액 [          ] 만원
  ────────────────────────
  상세 입력 >    (신용/체크/현금 따로)
```

총액만 입력하면 자동으로 신용:체크:현금 = 6:3:1 비율 추정 적용.
"상세 입력"을 누르면 7종 분리 화면 진입.

### 4-2. 자동 계산 강화

| 항목 | 자동화 방법 |
|------|-----------|
| 4대보험 | 연봉에서 자동 계산 (현재도 auto) |
| 식대 비과세 | 기본 ON, 토글로 OFF만 가능 |
| 육아보육수당 | 프로필(자녀 나이)에서 자동 |
| 부양가족 공제 | 프로필(부양가족 수)에서 자동 |
| 경로우대 | 프로필(만 70세 이상)에서 자동 |
| 자녀세액공제 | 프로필(만 8세 이상 자녀)에서 자동 |

**원칙**: 프로필에서 이미 답한 정보는 다시 묻지 않는다.

### 4-3. 스마트 기본값

카드 총액만 입력했을 때의 비율 추정:

```javascript
// 기본 추정 비율 (통계청 2024 가계동향조사 기반)
credit: 60%   // 신용카드
debit: 25%    // 체크카드
cash: 10%     // 현금영수증
transport: 5% // 대중교통
```

의료비 총액만 입력했을 때:
```javascript
general: 90%           // 일반 의료비
self_senior_disabled: 10%  // 본인/65세이상/장애인
// 나머지: 0 (해당 시 상세 입력 유도)
```

## 5. 네비게이션 정비

### 현재 문제

```
온보딩: goScreen('screen-q1') → onclick으로 하드코딩
탭 전환: switchTab('tab-home') → 별도 함수
바텀시트: openAuthSheet() / closeAuthSheet() → 별도 함수
설정 저장: saveSettings() → 페이지 리로드
```

뒤로가기 동작이 제각각. 브라우저 뒤로가기 누르면 앱을 이탈함.

### 변경안

**규칙 1**: 모든 화면 전환은 history stack에 push

```javascript
// 모든 화면 전환을 하나의 함수로 통일
function navigate(screenId, options) {
  history.pushState({ screen: screenId }, '', '#' + screenId);
  showScreen(screenId, options);
}

window.onpopstate = function(e) {
  if (e.state && e.state.screen) {
    showScreen(e.state.screen, { direction: 'back' });
  }
};
```

**규칙 2**: 화면 유형별 닫기 패턴 통일

| 화면 유형 | 닫기 방법 | UI 요소 |
|----------|----------|---------|
| 온보딩 스텝 | 좌상단 <- | back-btn (현재 있음) |
| 탭 화면 | 하단 탭바 | tab-bar (유지) |
| 바텀시트 | 오버레이 탭 or 아래로 스와이프 or X 버튼 | 3가지 모두 지원 |
| 상세 입력 | 좌상단 <- 또는 "완료" | 화면 상단에 항상 표시 |
| 설정 | 좌상단 <- | 자동 저장 (저장 버튼 제거) |

**규칙 3**: 하단 탭바 항상 표시 (온보딩 제외)

현재 `mainTabBar`는 `display:none`으로 시작해서 온보딩 후에만 보임. 이건 유지하되, 메인 진입 후에는 모든 화면에서 탭바 유지.

## 6. 고급 기능 숨기기

### 현재: 설정 탭에 모두 노출

```
설정
├── 기본 정보 (연봉, 원천징수비율)
├── 프로필 (결혼, 자녀, 주택, 세대주, 부양가족)
├── 가족 정보 (배우자 연봉, 차량)
├── 중도입사/퇴사          ← 고급
├── 복수 근무처              ← 고급
├── 전 직장 원천징수영수증    ← 고급
├── 알림
└── 저장하고 다시 계산
```

### 변경: 2단 분리

**설정 기본 화면:**
```
설정
├── 기본 정보 (연봉, 원천징수비율)
├── 프로필 수정 → (온보딩 플로우 재진입)
├── 알림
└── 고급 설정 >
```

**고급 설정 (하위 화면):**
```
고급 설정
├── 배우자 연봉 (맞벌이 최적화용)
├── 차량 소유
├── 중도입사/퇴사
├── 복수 근무처
├── 전 직장 원천징수영수증
└── 비과세 수당 상세 설정
```

### 비과세 항목도 분리

현재 비과세 항목 12종(식대, 육아보육수당, 자가운전보조금, 출산지원금, 야간근로수당, 연구보조비, 국외근로소득, 학자금, 일직숙직비, 벽지수당, 위험수당)이 공제 탭의 아코디언에 섞여 있음.

**변경**:
- 식대, 육아보육수당은 자동 적용 (토글 불필요)
- 자가운전보조금은 차량 설정에서 자동 연동
- 나머지 특수 비과세(야간근로, 연구비, 국외근로, 벽지, 위험 등)는 고급 설정으로 이동

### 8-5. 온보딩 (프로필 각 단계)

현재 구조 유지하되 개선:

```
Step 1: 연봉 입력         (현재 screen-salary)
Step 2: 결혼 여부          (현재 screen-q1)
Step 3: 자녀 여부 + 수     (현재 screen-q2)
Step 4: 주택 수            (현재 screen-q3)
  └ 4a: 세대주 여부        (현재 screen-q3b)
  └ 4b: 거주 형태          (현재 screen-q3c)
  └ 4c: 대출 여부          (현재 screen-q3d)
Step 5: 부양가족           (현재 screen-q4)
Step 6: 매칭 결과          (현재 screen-matched)
```

**변경점**:
- 프로그레스 바의 분모를 "5단계"로 통일 (현재 주택 서브스텝이 3/4, 3/4, 3/4로 반복됨)
- 뒤로가기 시 입력값 유지 (현재도 유지되나, history API 연동 필요)
- 주택 0채(무주택) 선택 시 세대주/거주형태/대출을 한 화면에 합치기 (3스텝 → 1스텝)

## 9. 구현 우선순위

### Phase 1: 즉시 적용 가능 (코드 변경 소)

1. **용어 변환** - label/desc 텍스트만 교체
2. **비과세 항목 숨기기** - 식대/육아보육 자동화, 나머지 고급 설정으로
3. **카테고리명 제거** - 아코디언 헤더의 "특별소득공제" 등 제거, 항목만 나열
4. **설정 2단 분리** - 고급 설정 버튼 추가, 하위 항목 이동

### Phase 2: 구조 변경 (중간 작업량)

5. **공제 탭 레이어 구조** - 기본 6개 + "더 보기" 패턴
6. **총액 우선 입력** - 카드/의료비/교육비/기부금/보험료에 총액 입력 추가
7. **바텀시트 상세 입력** - 상세 입력 시 바텀시트로 전환

### Phase 3: 네비게이션 (큰 작업량)

8. **History API 연동** - 모든 화면 전환에 pushState
9. **온보딩 스텝 최적화** - 주택 서브스텝 병합
10. **자동 저장** - 설정 탭 저장 버튼 제거, 입력 즉시 반영

## 11. 비과세 항목 재분류

현재 공제 탭에 비과세 12종이 소득공제/세액공제와 섞여 있음. 이것이 복잡도의 주요 원인.

### 자동 처리 (사용자 입력 불필요)
- 식대 (월 20만원) - 기본 ON
- 육아보육수당 - 프로필 자녀 나이에서 자동
- 자가운전보조금 - 차량 설정에서 자동

### 고급 설정으로 이동
- 출산지원금
- 야간근로수당
- 연구보조비/연구활동비
- 국외근로소득
- 학자금
- 일직/숙직비
- 벽지수당
- 위험수당

**이 변경만으로 공제 탭에서 12개 항목이 사라짐.**
